# his-platform 能力清单

> 医疗信息系统(HIS + EMR)微服务作品集。定位:展示 **DDD 领域建模、Spring Cloud 微服务全家桶、FHIR R4 医疗标准、医嘱状态机、事件驱动一致性、治理与可观测、多模式鉴权(含 SSO)**。
>
> 技术基线:Java 21(虚拟线程)· Spring Boot 3.3.5 · Spring Cloud 2023 · Spring Cloud Alibaba(Nacos/Sentinel)· PostgreSQL(DB per service)· Redis · Kafka(KRaft)· HAPI FHIR · Spring StateMachine · React 18 + TS + Vite + Ant Design。

## 二、鉴权与单点登录(SSO)能力 ⭐

网关支持**三种鉴权模式,按 Spring profile 共存切换**,互不干扰:

| 模式(profile) | 令牌 | 登录方式 | 网关校验 | 身份来源 |
|------|------|---------|---------|---------|
| **password**(默认) | HS256 自签 JWT | 账号密码(his-auth) | 手工验签(`AuthGlobalFilter`) | JWT claims |
| **idp** | RS256 | OIDC 授权码 + PKCE(his-idp) | OAuth2 Resource Server + JWK | 按 username 回查 his-auth |
| **dual** | **HS256 + RS256 双令牌** | 账号密码 **与** SSO 共存 | 多 issuer 校验(按 `iss` 分派) | HS256 读 claims / RS256 回查 |

**核心亮点**:

- **his-idp**:Spring Authorization Server 实现标准 **OIDC 授权码 + PKCE** 流,签发 RS256;暴露 `/.well-known/openid-configuration`、`/oauth2/authorize|token|jwks`。用户/口令仍以 his-auth 为单一事实来源(登录经 Feign 委托 `/auth/internal/verify` 核验,IdP 不落口令)。
- **网关作 Resource Server**:经 IdP 的 **JWK(RS256)** 验签,校验通过后按 token 的 `preferred_username` 回查 his-auth 换本地 `uid/dept/roles`(reactive `WebClient` + 60s TTL 缓存),再透传 `X-His-*` —— **下游服务零改动**。
- **dual 双令牌**:`JwtIssuerReactiveAuthenticationManagerResolver` 按 token 的 `iss` claim 分派 —— `his-auth`→HS 共享密钥解码器、`idp`→RS256 JWK 解码器,实现账号密码 + SSO 并存。
- **前端三模式**:`VITE_AUTH_MODE` = password / idp / dual;dual 模式一页两入口(密码表单 + SSO 按钮),登录方式记录后登出分流(SSO→IdP 端 RP-initiated 登出、密码→本地清理)。
- **RBAC**:网关透传 `X-His-*` → 下游 `UserContextInterceptor` 装配 `UserContext` → `@RequiresRole` 切面鉴权;下游信任内网网关,不重复验签。

**真机验证**:idp 授权码流 9/9、dual 双链路 8/8 断言全过(容器栈端到端)。

## 三、业务能力(按领域)

### 1. 主数据 · 患者主索引(his-mdm,9002)
- 患者建档 EMPI(企业级主索引),`gender` 必填枚举(MALE/FEMALE/UNKNOWN)
- 数据字典(按类型查)
- 端点:`POST /mdm/patients`(ADMIN/NURSE)、`GET /mdm/patients/{id}`、`GET /mdm/dicts/{type}`

### 2. 挂号 · 排班号源(his-registration,9003)
- 医生排班开号、号源查询、患者挂号、就诊标记
- **Redis Lua 原子锁防超挂**(高并发下号源不超卖)
- 端点:`POST /reg/schedules`(ADMIN)、`GET /reg/schedules`、`POST /reg/appointments`、`PUT /reg/appointments/{id}/visited`;`period` 必填(AM/PM)

### 3. 门诊就诊 + 医嘱(his-outpatient,9004)
- **充血领域模型** + **Spring StateMachine 医嘱状态机** + Kafka 事件
- 就诊(Encounter)开单、下诊断、开医嘱、提交、撤销/停止/重提交
- **医嘱按类型路由**:DRUG(药品)送药师审方,EXAM/TREATMENT 直达提交
- 端点:`POST /outpatient/encounters`、`PUT .../diagnosis`、`POST .../orders`、`POST .../submit`、`.../resubmit`、`.../cancel`(均 DOCTOR)

### 4. 药师审方(his-outpatient · Step 6)
- 处方审核状态机(`PENDING_REVIEW` / `REJECTED`),留痕表 `prescription_review`
- 计费守卫推迟:仅当就诊无待审/驳回医嘱时才发一次计费事件
- 端点:`GET /outpatient/reviews/pending`、`POST .../{encounterId}/pass|reject`(均 PHARMACIST)

### 5. 收费结算(his-billing,9006)
- **事件驱动计费**:消费 `his.order.placed` 生成账单
- 缴费,幂等以 `encounterId` 为键(账单按 encounterId upsert)
- 端点:`GET /billing/invoices/{id}`、`GET .../by-encounter/{encounterId}`、`POST .../{id}/pay`(CASHIER/ADMIN)

### 6. 退费审批流(his-billing · Step 6)
- **`RefundRequest` 充血状态机**:REQUESTED → APPROVED / REJECTED
- 发起(收费员)→ 审批(管理员);批准时 `Invoice.refund()` 并发 `RefundReviewed` 事件驱动 outpatient 撤销已执行医嘱
- 端点:`POST /billing/invoices/{id}/refund-requests`(CASHIER)、`GET /billing/refund-requests/pending`(ADMIN)、`POST .../{id}/approve|reject`(ADMIN)

### 7. 电子病历 + FHIR R4(his-emr,9005)
- 病历文档生成(消费缴费事件自动生成),按 `encounterId` 幂等 upsert
- **HAPI FHIR R4 输出**:`GET /fhir/Patient/{id}`、`GET /fhir/Encounter/{id}`(标准 `application/fhir+json`)
- 输出资源:Patient / Encounter / Composition / Condition / MedicationRequest
- 端点:`POST /emr/documents/{encounterId}`(DOCTOR)、`GET /emr/documents`

### 8. 通知 · 待办中心(his-notify,9007)
- **事件驱动 read-model**:消费各域事件落地通知
- 收件人二选一:`recipientRole`(角色待办,全员可见)或 `recipientUserId`(个人消息);收件箱按 `userId OR role IN roles` 过滤
- 端点:`GET /notify/inbox`、`GET /notify/unread-count`、`POST /notify/{id}/read`、`POST /notify/read-all`

## 四、事件驱动一致性(Saga)

事件 DTO 放 `his-api/event`,主题常量在 `Topics`,Kafka 传递(`spring.json.trusted.packages` 白名单):

```
门诊下医嘱 ──his.order.placed──▶ 收费生成账单
收费缴费   ──his.invoice.paid──▶ 门诊推进医嘱执行 + 病历生成
门诊送审   ──his.rx.review.requested──▶ 通知(药师审方待办)
药师审方   ──his.rx.reviewed──▶ 通知(审方结果)
退费发起   ──his.refund.requested──▶ 通知(退费待办)
退费审批   ──his.refund.reviewed──▶ 通知 + 门诊撤销已执行医嘱
```

- **幂等**:跨服务以 `encounterId` 作幂等键(账单、病历按 encounterId upsert)
- **计费防重**:`encounter.billed` 标志 + billing 侧 encounterId 唯一约束双重防重
